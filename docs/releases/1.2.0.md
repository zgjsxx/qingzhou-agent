# xu-agent v1.2.0

## 版本说明

1.2.0 进入开发阶段。本文件用于持续记录该版本新增功能、行为变更、问题修复和升级注意事项。

## 变更记录

- 将项目根版本、后端 Python 包、前端应用及 MCP 客户端版本统一更新为 `1.2.0`。
- 将前端字体从构建时下载的 Google Inter 改为项目内置 Geist 字体，支持完全离线构建。
- 优化本地启动顺序：等待 LangGraph `/info` 就绪后再启动前端，并为前端连接检查增加短暂重试，避免启动竞态产生误报。
- 后端启动默认禁用自动打开 LangSmith Studio 浏览器页面，需要时仍可手动访问。
- 前端 Plugins 面板支持启用/禁用已配置的 MCP 插件，并提示需要重启后端后生效。
- 修复 `run_ssh_command` 在远端命令读取 stdin 时容易挂起直到超时的问题：SSH 调用改为显式非交互执行，关闭 stdin 转发，并在超时时清理进程树、返回已捕获输出。
- 修复前端左侧本地面板中“插件”“运行监控”菜单项的中文乱码问题，并在源码中保留编码规避说明。
- 新增 `/compact [focus]` 手动上下文压缩命令：命令由后端上下文中间件拦截，不交给主模型执行，并复用自动压缩的摘要、消息替换、metadata 与日志更新逻辑；手动压缩默认尽量把历史消息都压入摘要，可通过 `AGENT_MANUAL_COMPACT_KEEP_MESSAGES` 调整保留的最近原文数量。
- 新增 `/help` 斜杠命令：后端直接返回确定性的命令清单，不交给主模型生成，避免帮助内容和实际支持的命令不一致。
- 统一斜杠命令入口：`agent_commands.py` 负责解析 `/help`、`/clear`、`/compact`，自有 UI、飞书和后端图中间件复用同一套命令模型，减少新增命令时的分散改动。
- 新增基于 `qq-botpy` 的腾讯 QQ 机器人桥接层：支持频道 `@机器人` 与私信文本消息接入，复用现有 LangGraph graph、历史消息、日志与 slash command 分发，并预留群 `@` / C2C 事件处理入口。
